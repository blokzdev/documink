# ADR-001: Flutter as cross-platform framework

- Status: Accepted
- Date: 2026-06-13
- Context: Recorded in V0 Phase 3; formalizes a decision already in the specs.

## Context

DocuMink targets Android first (V1), then Windows (V2), with iOS/macOS deferred past V4
(ADR-015). It is a UI-rich, local-first app needing camera, biometrics, file handling, and an
on-device ML/LLM stack. We need one framework that delivers native performance on phones and
desktops from a shared codebase, with a permissive license.

## Decision

Build the app in **Flutter (≥3.38) / Dart (≥3.10)**, stable channel. Reference toolchain is
Flutter 3.38.6 / Dart 3.10.7. Riverpod (≥2.5) for state, go_router (≥14) for routing.

## Rationale

- Single codebase spans the Android→Windows roadmap; Flutter has first-class desktop support.
- Native compilation (no JS bridge) meets the cold-start (<2 s) and detection-latency targets.
- BSD-3 licensed — clean under the project license policy (ADR-008/`license-policy.md`).
- Mature plugin ecosystem for camera, `local_auth`, secure storage, and FFI to native ML runtimes.

## Consequences

- Web is explicitly out of scope V1–V4 (blueprint §1.2); platform channels carry OS-specific
  bits (Keystore/DPAPI, Windows AI APIs).
- The stack floor (Flutter ≥3.38 / Dart ≥3.10) is a hard minimum enforced in CLAUDE.md and CI.
- Dart 3.10 build-hooks force `--force-jit` for codegen (`dart-toolchain.md`).
