# ADR-011: Device-capability-tiered Tier 4 LLM with Balanced/Specialized variants, signed remote tier catalog, transparent floor UX, opt-in upper tiers, no silent swaps

- Status: Accepted
- Date: 2026-06-13
- Context: Recorded in V0 Phase 3; formalizes a decision already in the specs.

## Context

Tier 4 is the optional on-device LLM powering Mink and deep detection. DocuMink targets hardware
from 2 GB-RAM phones to 32 GB workstations. A single model can't fit that range, and we must
never surprise users by changing the model under them or auto-downloading huge files.

## Decision

Adopt a **device-capability-tiered Tier 4 LLM** system:

- A **Device Capability Profiler** computes a numeric score (RAM, storage, CPU, NPU, GPU, system
  model, form factor) and selects the highest qualifying tier from a **remote, Ed25519-signed
  tier catalog**, with a 20% free-storage headroom check.
- Each tier offers a **Balanced (auto-selected)** and optional **Specialized (user-switchable)**
  variant; Minimum and System-provided are single-option.
- **Upper tiers (Professional/Workstation) are opt-in only** and never auto-recommended; the
  auto-recommend ceiling is Performance (6 GB), floor is Minimum (2 GB).
- Below the floor, a **transparent UX** disables Tier-4 affordances (visible, not hidden) while
  Tiers 1–3 keep working.
- **No silent swaps** — any tier/variant/model-version change prompts a before/after comparison.

## Rationale

- Capability-based selection with no hard ceiling lets new tiers ship via signed manifest without
  app releases (blueprint §4.7).
- "No silent swaps" and Ed25519 verification are binding architecture invariants
  (`architecture-invariants.md`; blueprint §15 #15, #16; models.md §1).
- Gemini Nano / Phi Silica are opportunistic system-provided options, never a hard dependency
  (blueprint §15 #4).

## Consequences

- Runtime dispatch is model-agnostic behind an `LlmBackend` interface (GGUF→`fllama`,
  `.task`→`flutter_gemma`, system models via platform APIs) — ADR-002 separates encoders.
- The catalog, hosting, quantization, and signing details are authoritative in `models.md` (ADR-017).
- High-stakes territory (CLAUDE.md V1 Phase 9 — profiler + signed manifests).
