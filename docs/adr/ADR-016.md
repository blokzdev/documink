# ADR-016: Mink memory architecture — six typed stores adapted from MIRIX taxonomy with single-agent deterministic router, PII-safe reference model via vault token fingerprints, SQLite + sqlite-vec on-device

- Status: Accepted
- Date: 2026-06-13
- Context: Recorded in V0 Phase 3; formalizes a decision already in the specs.
- **Authoritative source: `docs/memory.md`.** This ADR summarizes; `memory.md` wins on any memory detail.

## Context

Mink needs persistent, personalized memory across sessions and paired devices, without ever
holding raw PII/PHI plaintext, and scaled to device capability — on the single-agent design of
ADR-012 and the single encrypted database of ADR-003.

## Decision

Adopt a **six-type memory taxonomy** adapted from MIRIX:

1. Core, 2. Episodic, 3. Knowledge Vault (the existing `tokens` table) — **active V1**;
4. Semantic, 5. Procedural, 6. Resource — **schema V1, activation V1.2**.

Key properties:
- **PII-safe reference model (critical invariant):** memory rows never store plaintext PII —
  only token references (`{type:token_ref,…}`, `<<tok_…>>`) or HMAC-SHA256 `canonical_fingerprint`
  values pointing into the vault. `MemoryRepository.write` rejects unreferenced PII.
- **Single-agent deterministic router** (per ADR-012), not multi-agent.
- **Storage is the one SQLCipher database** with `sqlite-vec` for vector search and FTS5 for
  text — **not Postgres or any specialized DB** (ADR-003).
- **Project-scoped isolation** via repository-layer `workspace_id`/`project_id` (ADR-007).
- **Tier-scaled updates** (ADR-011): high tiers do autonomous structured updates; low tiers do
  explicit-only.

## Rationale

- Reusing the vault's `tokens` table as the Knowledge Vault means the vault's security guarantees
  extend to memory — the memory DB can be inspected without revealing PII (memory.md §1.1, §3).
- Fingerprint-based identity lets Mink reason about entities without materializing plaintext
  (memory.md §3.2).

## Consequences

- Adding a seventh memory type, or writing plaintext PII to any memory table, is forbidden
  (memory.md §12; blueprint §15 #24).
- Full schemas, retrieval patterns, router, sync semantics, and threat model live in `memory.md`.
- High-stakes territory (CLAUDE.md V1 Phase 12).
