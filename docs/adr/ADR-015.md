# ADR-015: iOS/macOS deferred until after V4

- Status: Accepted
- Date: 2026-06-13
- Context: Recorded in V0 Phase 3; formalizes a decision already in the specs.

## Context

DocuMink is built on Flutter (ADR-001), which can target Apple platforms. But the on-device
ML/LLM stack, key management, and platform integrations are deep per-platform investments, and
the roadmap prioritizes Android (V1) then Windows (V2). Spreading thin across Apple platforms
early would slow the core roadmap.

## Decision

**Defer iOS and macOS until after V4.** The platform priority is Android (V1) → Windows (V2) →
expansion/document-platform (V3/V4), with iOS/macOS *reconsidered* post-V4 and Linux/Web out of
scope (Web never in V1–V4).

## Rationale

- Focus engineering on Android then Windows to ship a complete product before adding platforms
  (blueprint §1.2; PRD §3).
- Apple platforms bring distinct runtime/sandboxing and distribution constraints best tackled
  once the architecture has matured.

## Consequences

- Apple-specific work (e.g. iCloud BYOC) is explicitly post-V4 (blueprint §9.5).
- The architecture stays portable (Flutter, shared ONNX artifact, `LlmBackend` abstraction) so a
  future Apple port is feasible without a rewrite.
