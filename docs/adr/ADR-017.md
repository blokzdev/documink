# ADR-017: Tier 4 model hosting — Play Asset Delivery primary for Android V1, HuggingFace direct for Windows V2, Ed25519-signed `manifest.json` on documink.ai as the only self-hosted artifact

- Status: Accepted
- Date: 2026-06-13
- Context: Recorded in V0 Phase 3; formalizes a decision already in the specs.
- **Authoritative source: `docs/models.md`.** This ADR summarizes; `models.md` wins on any Tier 4 detail.

## Context

Tier 4 models (ADR-011) range from ~220 MB to ~18 GB and are downloaded on demand. For a freemium
app with uncertain growth, paying egress to self-host them is infeasible — yet downloads must be
integrity-verified and updates must never be silent.

## Decision

Host Tier 4 models per platform, with one signed control file:

- **Android V1 → Google Play Asset Delivery (PAD)** — Google absorbs bandwidth; Standard-Balanced
  is an install-time pack (skippable), all others on-demand; files >1 GB split into sub-packs.
- **Windows V2 → direct HuggingFace downloads** of the exact quantized file, resume-capable.
- **documink.ai hosts only the Ed25519-signed `manifest.json`** (few KB) — the single source of
  truth the app reads — plus cold emergency mirrors that are not a primary path.
- **SHA-256 pinning** on every model file; **Ed25519 signature verification** on every manifest
  fetch (failure aborts the update, never falls back to unsigned); **no silent swaps**.
- **Quantization is one-per-family:** Gemma 4 → LiteRT `.task` int4 QAT; Qwen/SmolLM/Phi → GGUF
  `q4_k_m`. Every catalog model is Apache 2.0 or MIT.

## Rationale

- PAD is cost-free and Play-signed; HuggingFace already hosts these models for Windows; only the
  tiny manifest must be ours (models.md §1, §2).
- Manifest-driven distribution lets tiers update without app releases while keeping supply-chain
  integrity (models.md §5, §6).

## Consequences

- Do not host primary model distribution on documink.ai for Android (blueprint §15 #29; models.md §9 #5).
- Do not hard-code model URLs in Dart — they come from the signed manifest (models.md §9 #6).
- Any Tier 4 model add/change must update `models.md` in the same PR (`architecture-invariants.md`).
- CI verifies model hashes (`tool/scripts/verify_model_hashes.dart`); high-stakes (CLAUDE.md V1 Phase 9).
